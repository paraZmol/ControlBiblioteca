por requerimiento de seguridad y control administrativo, vamos a eliminar el acceso híbrido. El sistema debe ser "Netamente Local". Realiza los siguientes cambios:

1. Lógica de Validación (Backend):

Modifica el endpoint de login de la terminal para que únicamente busque el DNI en la tabla alumnos_maestro.

Elimina por completo la llamada a la función que consulta el API externo del SGA. Ya no queremos auto-registros automáticos.

2. Manejo de Errores:

Si el DNI no existe en la base de datos local, devuelve un error claro a la terminal: "Usuario no registrado. Acérquese al módulo para tramitar su carnet de biblioteca".

3. Preservación de Datos:

Asegúrate de que este cambio no afecte la tabla de sesiones. Los alumnos que ya están registrados (como los que importamos o los de prueba que ya existen) deben poder seguir entrando sin problemas.

El objetivo es que el lunes, si un alumno no está en el Excel que cargamos, el sistema le deniegue el acceso de inmediato.
